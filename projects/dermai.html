Overview

DermAI is a comprehensive dermatological triage system that combines deep learning image classification with retrieval-augmented generation (RAG) for conversational health guidance. Upload a skin photo, receive instant predictions across 23 condition categories, then chat with an AI assistant that grounds its medical advice in evidence-based sources.

The classification backend uses a DenseNet121 convolutional neural network trained on the DermNet dataset (19,000+ dermatoscopic images). The model learns hierarchical patterns from low-level textures to high-level lesion morphologies to identify conditions ranging from melanoma to benign nevi.

The RAG-powered chatbot bridges the gap between algorithmic predictions and human understanding. By embedding medical documentation into a vector database (Supabase pgvector), the system retrieves relevant context for user questions and synthesizes responses using Gemini 2.0 Flash. Every answer cites sources, ensuring transparency and medical accuracy without hallucination.

Technical Deep Dive

Image Classification Architecture

The classification backbone uses DenseNet121, a convolutional neural network optimized for feature reuse through dense connections. Trained on the DermNet dataset (19,000+ dermatoscopic images), the model learns hierarchical patterns from low-level textures to high-level lesion morphologies to classify skin conditions into 23 categories.

The training strategy employs data augmentation (rotation, flipping, color jittering), class weighting to handle imbalance, and early stopping to prevent overfitting. The final model achieves robust generalization across diverse skin tones and imaging conditions.

RAG System Architecture

1. Knowledge Ingestion

Medical documentation chunked and embedded using OpenAI's text-embedding-3-small

2. Vector Search

Supabase pgvector stores embeddings, enabling similarity-based retrieval

3. Context Augmentation

Relevant medical facts injected into prompt for grounded responses

4. LLM Generation

Gemini 2.0 Flash synthesizes evidence-based answers with source attribution

Full-Stack Implementation

  • Frontend: Next.js 15 with Tailwind CSS for responsive UI, shadcn/ui for accessible components
  • Backend: Flask API for image classification (PyTorch model serving), Next.js API routes for RAG orchestration
  • Database: Supabase PostgreSQL with pgvector extension for embeddings + user authentication
  • AI Orchestration: Vercel AI SDK for streaming chat responses, tools system for dynamic function calling
  • Authentication: Supabase Auth with Google OAuth, session management via middleware

Implementation Highlights

Classification Endpoint

Flask API exposes /predict endpoint that accepts image uploads, preprocesses to 512Ã—512 resolution, runs inference through PyTorch-loaded DenseNet121, and returns probability distribution across 23 categories. Model weights hosted separately, loaded on server startup to minimize cold-start latency.

RAG Chatbot Tool System

The chatbot uses Vercel AI SDK's tool system for dynamic function calling:

Classification Tool: Fetches prediction results from user's uploaded image when medical diagnosis is requested

Knowledge Tool: Triggers semantic search across vector database when user asks specific medical questions

Response Synthesis: Gemini 2.0 Flash combines tool outputs with conversation context to generate grounded, cite-tagged responses
