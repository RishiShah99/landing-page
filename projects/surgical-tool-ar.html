MIXED REALITY MEDICAL PROTOTYPE

SurgicalToolAR

Proximity Aware AR System for Surgical Training
🔬 Unity + Vuforia 📱 Android Tablet Demo ⚡ Real Time Tracking

Overview

SurgicalToolAR is a marker based augmented reality prototype that tracks a 3D brain model and a physical pen as a simulated surgical probe. The system calculates real time distance measurements in millimeters and displays dynamic safety warnings (SAFE, CAUTION, DANGER) based on proximity thresholds. This goes beyond basic AR visualization by adding spatial awareness between tracked instruments and anatomical targets.

Most AR medical demos stop at "here's a brain floating on a marker." That looks impressive in presentations, but it's not actually useful in clinical settings. Real surgical guidance needs interaction awareness: Where is the tool? How close is it to critical tissue? Can we give meaningful real time feedback?

This prototype demonstrates the next step. Instead of just displaying anatomy, it tracks tool position relative to the 3D model, computes proximity in real time, and provides safety zone feedback. The system includes touch controls for rotation, scaling, locking, and measurement modes, creating a functional proof of concept for proximity aware surgical training.

The Problem

Mixed reality surgical visualization faces a fundamental limitation: displaying anatomy in 3D space is the easy part. The hard part is making it interactive and clinically meaningful.

Specifically, you need:

  • Spatial awareness: Where is the surgical tool positioned relative to the anatomy?
  • Proximity detection: How close is the instrument to critical tissue or target regions?
  • Real time feedback: Can the system provide simple, actionable warnings like safe, caution, or danger zones?

Without these features, AR surgical demos remain impressive visualizations but lack practical utility. This project addresses that gap by building proximity aware interaction into the AR experience.

The Solution

I built a marker based AR prototype using Unity and Vuforia that combines anatomical visualization with real time tool tracking and proximity analysis.

Core Features

  • Anchors a 3D brain model to a printed base marker for stable AR positioning
  • Tracks a physical pen (with marker fins) as a surrogate surgical tool in 3D space
  • Calculates real time distance from tool tip to nearest brain surface using collision geometry
  • Displays distance measurements in millimeters with color coded safety zones
  • Provides touch controls for rotation, scaling, interaction locking, and measurement mode
  • Runs as an Android tablet APK for portable demonstration and testing

The result is a system that doesn't just show anatomy, it tracks tool approach and gives meaningful spatial feedback, which is much closer to what actual surgical guidance systems need to do.

Technical Implementation

AR Anchoring with Vuforia

The system uses three image targets:

  • Base marker: Tracks the 3D brain model position and orientation (BrainTarget)
  • Tool markers: Front and back tracking markers attached to a physical pen (FrontTarget, BackTarget)

Vuforia provides 6DOF pose estimation (position and rotation) for each target, allowing the system to maintain accurate spatial relationships even as the camera or markers move.

Brain Interaction Controls

I implemented multi touch gesture controls for manipulating the AR brain:

  • PinchScale: Two finger pinch gestures for zooming in and out
  • DragRotate: Single finger drag for free rotation around axes
  • LockController: Toggle to freeze all interactions for stable measurement
  • MeasureModeController: Locks scale to ensure consistent distance calculations

Tool Tracking System

The tracking rig handles marker based tool pose estimation:

  • Dual marker setup: Front and back markers provide redundancy and orientation
  • ToolRigFollower script: Intelligently selects the best visible marker and smoothly follows its pose
  • Marker loss handling: Manages tracking dropout with smooth interpolation and reacquisition
  • Jitter reduction: Applies temporal smoothing to reduce snapping and visual noise

Proximity Detection Logic

This is where the system becomes clinically relevant:

  • Tool tip representation: A dedicated transform marks the precise point of the simulated probe
  • Collision approximation: The brain is represented using capsule and sphere colliders for fast distance queries
  • Distance calculation: Uses Unity's ClosestPoint() to find the nearest surface point from the tip to the brain geometry
  • Safety thresholds: Distance values are classified into SAFE, CAUTION, and DANGER zones based on predefined thresholds
  • Smoothing and hysteresis: Prevents status flickering by requiring sustained threshold crossing before changing safety states

The output is a real time distance measurement (in mm) and a color coded safety indicator displayed in the AR HUD.

User Interface

The HUD provides clean, readable feedback without cluttering the AR view:

  • Bottom left buttons: Reset, Lock, and Measure mode controls
  • Bottom right status panel: Large distance readout and safety zone indicator
  • Visual design: Transparent black background with high contrast text for outdoor/variable lighting
  • Color coding: Green for safe, yellow/orange for caution, red for danger zones

What This Accomplishes

The key achievement here is demonstrating that AR surgical visualization can move beyond passive 3D models to active spatial interaction systems.

This is not just: "AR model on a marker"

This is: AR anatomy + tracked instrument + live distance feedback + safety state classification

That difference makes this a significantly stronger prototype for medical extended reality applications compared to typical classroom AR projects. It shows an understanding of what clinical utility actually requires, proximity awareness, safety feedback, and robust tracking under real world conditions.

Potential Applications

Current Prototype Use Cases

  • Medical education: Teaching students safe approach angles and proximity awareness with anatomical models
  • Surgical simulation demos: Showcasing how mixed reality can provide safety overlays during procedures
  • Human factors research: Testing whether visual proximity warnings improve precision in controlled studies

Future Development Paths

If I were to continue this project, here are the logical next steps:

  • Anatomical region segmentation: Replace simple colliders with detailed segmented regions (tumor, vessels, cortex, critical zones) for region specific warnings
  • Tool specific calibration: Support different surgical instrument geometries with unique tip offsets and safety thresholds per tool type
  • Markerless tracking: Migrate from printed markers to HoloLens or SLAM based world tracking for hands free operation
  • Multi region proximity: Track distance to specific anatomical structures, not just the whole brain, enabling warnings like "3mm from tumor boundary"
  • Trajectory logging: Record tool paths, minimum distances, and warning events for post procedure analysis and training feedback

Technical Takeaways

Beyond the AR implementation itself, this project taught me several valuable lessons about building spatial computing applications:

  • Marker based tracking is robust: Vuforia's image targets work remarkably well for prototyping, even with occlusion and variable lighting
  • Smoothing is critical: Raw tracking data is too noisy for direct use. Temporal smoothing and hysteresis make a huge difference in user experience
  • Collision approximation works: You don't need perfect mesh distance fields. Capsule and sphere colliders are fast enough for real time feedback and accurate enough for training scenarios
  • UI matters in AR: Transparent overlays, large text, and status color coding are essential for readability in mixed reality environments
  • Clinical context drives design: Understanding what surgeons actually need (proximity, not just visualization) shaped every technical decision in this project

Conclusion

SurgicalToolAR demonstrates that augmented reality for medical applications needs to go beyond visualization and provide actionable spatial awareness. By combining marker based tracking, real time proximity detection, and safety zone feedback, this prototype shows a path toward clinically meaningful mixed reality surgical guidance systems.

The next evolution of this work would involve headset deployment (HoloLens), markerless tracking, and integration with actual surgical instrument tracking systems. But as a proof of concept for proximity aware AR interaction, it accomplishes exactly what it set out to do: show that we can build systems that not only display anatomy, but actively understand and respond to tool positioning in real time.
